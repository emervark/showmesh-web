# Keyboard shortcuts

Use <kbd>Ctrl</kbd> on Windows. The code also supports <kbd>Cmd</kbd> for the
same modifier combinations.

## File and editing

| Shortcut | Command | In Show mode |
|---|---|---|
| <kbd>Ctrl</kbd>+<kbd>N</kbd> | New project | Locked |
| <kbd>Ctrl</kbd>+<kbd>O</kbd> | Open project | Locked |
| <kbd>Ctrl</kbd>+<kbd>S</kbd> | Save | Locked |
| <kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>S</kbd> | Save As | Locked |
| <kbd>Ctrl</kbd>+<kbd>Z</kbd> | Undo | Locked |
| <kbd>Ctrl</kbd>+<kbd>Y</kbd> | Redo | Locked |
| <kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>Z</kbd> | Redo | Locked |

## Transport and navigation

| Shortcut | Command |
|---|---|
| <kbd>Space</kbd> | GO when focus is not in an input |
| <kbd>Esc</kbd> | Close command palette; otherwise PANIC |
| <kbd>Delete</kbd> or <kbd>Backspace</kbd> | Delete selected cue in Edit mode |
| <kbd>Ctrl</kbd>+<kbd>K</kbd> | Toggle command palette |
| <kbd>Enter</kbd> | Commit a text or numeric field by blurring it |

::: danger Escape is PANIC
When the command palette is closed and focus is outside an input,
<kbd>Esc</kbd> is global PANIC. Do not use it merely to clear selection during
a live show.
:::
