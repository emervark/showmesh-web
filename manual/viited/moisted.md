# Glossary

| Term | Meaning |
|---|---|
| **Action** | A command or parameter change inside a cue |
| **Arming / standby** | Preparing a cue as the target of the next GO |
| **Cue** | One fireable show event |
| **Cue list** | Ordered list of cues |
| **Editor** | Electron/React user interface |
| **Engine** | C++ process that owns state and runs the show |
| **Follow** | Automatically firing the next cue after completion |
| **GO** | Fire the armed cue |
| **Headless** | Engine operation without the editor |
| **Layer** | Numeric visual stacking order |
| **NDI** | Network audio/video transport |
| **PANIC** | Global panic fade; a second press hard-stops |
| **Program** | Final audience/downstream composition |
| **Resource** | Media, text, timecode, or live-input source |
| **Show mode** | Edit-lock mode for live operation |
| **Spout** | Windows GPU texture sharing between applications |
| **Trigger** | Lifecycle, external, manual, or timecode event that fires an action |
| **Transition** | Parameter change over time |
