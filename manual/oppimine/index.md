# Learning

The core Showmesh model is simple: a **cue defines a lifecycle**, an **action
adds behaviour**, and a **trigger decides when the action fires**.

Learn these topics in order:

1. [Cues and the cue list](/oppimine/cued) — sequence, states, timing, and GO.
2. [Actions and triggers](/oppimine/actionid-ja-triggerid) — commands, values, and fades.
3. [Transitions](/oppimine/uleminekud) — smooth parameter changes and crossfades.
4. [Layers](/oppimine/kihid) — multiple simultaneous visuals and their order.

For exact allowed values, use the [Reference](/viited/).
