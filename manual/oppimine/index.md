# Learning

A **cue defines a lifecycle**, an **action adds behaviour**, and a **trigger
decides when it fires**.

1. [Cues and the cue list](/oppimine/cued) — arming, states, timing, timecode GO, and transport.
2. [Actions and triggers](/oppimine/actionid-ja-triggerid) — commands, values, fades, and external triggers.
3. [Transitions](/oppimine/uleminekud) — parameter motion and crossfades.
4. [Layers and compositing](/oppimine/kihid) — order, opacity, and geometry.

For allowed fields and values, use [Reference](/viited/).
